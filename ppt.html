微信小程序

--国信泰九|互联网研发部 杨磊磊

1. 使用场景，有何特色?

微信小程序是基于微信体系，无需下载安装就可以使用的应用，体积一般都不超过2M,是一个微信直接打开能用的小应用。于2017-01-09正式公诸于世面向开发者。

小程序的使用场景比如：发放优惠券、活动预约、共享单车、订外卖、看天气、安排日程…等等生活场景。

它无需安装、触手可及、用完即走、无需卸载。

2. 开发前你需要具备什么？

设计？

前端开发中度依赖UI，要求遵循微信Ui设计规范是个非常不错的选择。

开发工具？

微信有自己开发套件。但是在项目时间过程中，我并不推荐大家使用开发者工具开发，但是强烈推荐用它来调试。

成为开发者？

注册小程序账号，这个一般是公司的运营人员去操作，运营同学完成前期的注册后尽量要求运营同学暂时将管理员转让给前端负责人。小程序的开发人员的设置、上传代码、发布版本、域名服务配置都需要管理员的权限才能设置。

3.2 发起网络请求wx.requst()

  • 需要在小程序开发设置-服务器域名中配置后端服务。
  • 域名只支持https, 不能使用localhost和Ip地址。
  • 域名还必须ICP备案。
  • 请求并发数不超过5个。
  • 不过开发阶段可以再关闭域名验证。

3.3 维护登录信息

  1. wx.login() 获取code
  2. 使用code与后台交换session
  3. 后端接受code返回session给前端
  4. 采用localStorage存储session

3.4 如何获取用户的手机号码

通过getPhoneNumber。注意：目前该接口针对非个人开发者，且完成了认证的小程序开放。需谨慎使用，若用户举报较多或被发现在不必要场景下使用，微信有权永久回收该小程序的该接口权限。

3.13 项目工程的最后一步，如何部署？

  • 需要项目管理者的身份，在微信开发者工具上，在将代码打上版本信息上传到微信管理后台。
  • 然后有管理员登录后台进行操作。可以将版本设置成为体验版，邀请相关人员（产品、UI）进行相关测试。
  • 发现问题, 改正后重回步骤1， 直到所有问题解决后提交正式版本。填写相关表单信息，提交审核。需要审核大约15个工作日。

4. 总结

和我们平时开发项目上有哪些区别？

和传统页面上的开发而言，一个页面的创建显得很麻烦。不支持html直接编写，写的很膈应。 https://wepyjs.github.io/wepy小程序组件化开发框架。 类似vue组件的模式去写小程序。

思考小程序所适合场景。审核耗时很长以及小程序那些方面能做那些方面不能做。别用心做了写功能却不能通过审核。

开发前，思考公用模块、公用方法等等。
